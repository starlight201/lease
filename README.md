
# 公寓管理系统 (MyHouse) - 后端项目

## 项目简介
**公寓管理系统**是一个集移动端（面向租客）和后台管理系统（面向管理员）于一体的公寓租赁平台。系统旨在通过信息化手段提升公寓管理效率与服务质量，为租客提供便捷的找房、看房、签约体验，为管理员提供高效的房源、租约、用户管理工具。

## 系统架构
### 物理架构
系统采用经典的**客户端-服务器**架构。
1.  **客户端**: 用户通过浏览器（移动端或管理后台）访问系统。
2.  **应用服务器**: 接收客户端请求，处理业务逻辑，并与数据库交互。
3.  **数据库服务器**: 运行MySQL，持久化存储所有业务数据。

### 逻辑架构 (三层架构)
应用服务器内部采用三层架构设计，确保职责分离：
1.  **表示层 (Presentation Layer)**: 提供移动端和管理后台的API接口。
2.  **业务逻辑层 (Business Layer)**: 实现系统的核心业务规则和处理流程。
3.  **数据访问层 (Data Access Layer)**: 封装所有数据库的增删改查操作。

### 软件模块划分
系统后端代码按功能划分为以下顶层包：

 | 模块名称 | 职责描述 |
 | :--- | :--- |
 | **实体包 (Entity Package)** | 定义与数据库表对应的核心数据模型（实体类）。 |
| **业务包 (Business Package)** | 封装所有核心业务逻辑，是系统的心脏。 |
| **工具包 (Utility Package)** | 提供通用的技术组件和支持服务。 |

#### 业务包 详细模块
业务包具体由以下六个模块构成，处理不同领域的业务：

| 模块代号 | 模块名称 | 核心职责 |
| :--- | :--- | :--- |
| MYHOUSE-4.1 | 用户业务 | 租客注册、登录、个人信息管理、账号状态管理。 |
| MYHOUSE-4.2 | 管理员业务 | 后台管理员对异常用户、房源的审核，封禁操作，账单审批。 |
| MYHOUSE-4.3 | 房源业务 | 公寓及房间信息的增删改查(CRUD)、状态管理（上架/下架/已租）、房源检索。 |
| MYHOUSE-4.4 | 预约业务 | 看房预约的创建、取消、状态流转（如待确认/已完成）、预约冲突检测。 |
| MYHOUSE-4.5 | 租约业务 | 电子合同生成、租约续期、提前退租违约金计算、押金扣除规则。 |
| MYHOUSE-4.6 | 属性业务 | 房源相关属性（如配套设施、标签）的分类与管理。 |

#### 工具包 (`MYHOUSE-5`) 详细模块
工具包提供跨业务模块的通用技术支持：

| 模块代号 | 模块名称 | 核心职责与集成技术 |
| :--- | :--- | :--- |
| MYHOUSE-5.1 | 存储服务集成模块 | 基于 **MinIO** 实现文件（合同、图片）的上传、下载与存储管理。 |
| MYHOUSE-5.2 | 数据持久化支持模块 | 集成 **MyBatis-Plus**，提供自动填充、分页插件及通用CRUD接口。 |
| MYHOUSE-5.3 | 认证与安全控制模块 | 封装 **JWT** 令牌生成/验证、线程级用户信息管理、验证码生成。 |
| MYHOUSE-5.4 | 全局响应与异常模块 | 统一API响应格式(`Result`类)、自定义异常及全局异常拦截处理。 |
| MYHOUSE-5.5 | 第三方服务集成模块 | 对接 **阿里云短信API** 发送验证码、**Redis** 缓存管理会话与高频数据。 |

## 数据库设计
系统使用MySQL 8.0关系型数据库。主要数据表及其设计要点如下：

| 表名 | 存储内容 | 关键设计要点 |
| :--- | :--- | :--- |
| **用户信息表 (User)** | 租客账户信息。 | 密码加密存储(SHA-256)，邮箱/手机号建索引，HTTPS传输。 |
| **租约信息表 (Lease)** | 租赁合同信息。 | 与用户、房间表外键关联，敏感信息加密，状态字段索引。 |
| **浏览历史表 (BrowseHistory)** | 用户浏览房源记录。 | 关联用户与房间，按浏览时间索引，支持用户自主删除。 |
| **预约看房信息表 (Appointment)** | 看房预约记录。 | 关联用户与房间，预约时间字段索引，状态管理。 |
| **房间信息表 (Room)** | 房间详细信息。 | 关联公寓，价格、标签等字段索引，支持图片URL存储。 |
| **公寓信息表 (Apartment)** | 公寓详细信息。 | 包含社区、设施等描述，与房间表为一对多关系。 |
| **管理员账户信息表 (Admin)** | 后台管理员账户。 | 密码加密存储，权限管理基础。 |

**性能与扩展性考虑**:
- 为常用查询字段建立索引。
- 考虑通过分库、分表、分区来应对数据增长。
- 使用缓存技术（如Redis）减少数据库压力。

## API接口概览 (基于移动端需求推断)
后端为前端提供RESTful API，主要接口领域包括：
- **用户认证**: 登录、注册、发送验证码、修改密码。
- **房源服务**: 分页检索房源、获取房源/公寓详情、获取房源列表。
- **预约看房**: 创建、查看、取消预约。
- **租约管理**: 创建电子合同、查看我的租约、申请终止/延长租约、支付租金（对接支付宝/微信）。
- **个人中心**: 查看浏览历史、管理个人信息。

## 部署与运维
### 部署结构
项目预计打包为WAR或JAR包，部署在Tomcat 9.0应用服务器上，通过Nginx等反向代理提供服务。数据库独立部署。

### 系统维护与监控
- **日志记录**: 系统记录操作日志和运行日志，便于故障排查。
- **数据备份**: 定期进行数据库备份，并支持实时备份机制以实现快速恢复。
- **健康检查**: 具备运行环境自检机制（网络、磁盘空间等）。

### 错误处理与恢复
- **后备技术**: 设计数据库主从切换或备用服务器，以防止单点故障。
- **恢复策略**: 发生故障时，通过备份数据快速恢复，保证服务可用性。
